## 📊 L-CASH 项目 PPT 汇报结构建议

### 1. 项目愿景与问题定义 (Problem Statement)
* **内容**：不要直接说“我想做一个智能管家”，要说“针对目前智能家居碎片化、传感器数据噪声大、以及交互不自然的问题”。
* **专家技巧**：引用 PPT L06 中的 **User-Centered Design** 理念，说明你解决的是真实痛点（如：环境健康、独居安全）。

### 2. 系统架构：四层模型 (System Architecture) —— **[核心得分项]**
* **内容**：绘制一张精美的架构图，清晰标注：
    * **感知识别层**：Node 1-4 (M5Stack, Sensors).
    * **网络传输层**：MQTT (EMQX Broker), HTTP (ThingSpeak).
    * **平台支撑层**：Flask Backend, AI Agent (LLM), ThingSpeak Cloud.
    * **应用交互层**：Responsive Web UI.
* **专家技巧**：提到你对 **Edge vs Cloud** 的权衡（L05）。例如：Node 3 本地处理指纹和报警（Edge），而数据分析放在云端（Cloud）。

### 3. 多节点协同设计 (Multi-Node Collaboration)
* **内容**：重点展示 4 个节点如何各司其职又统一协作。
    * Node 1 & 2：环境感知。
    * Node 3：安全防护（ToF + 指纹）。
    * Node 4：交互执行（手势 + 风扇）。
* **专家技巧**：强调 **MQTT 协议的优越性**（发布/订阅模式、异步通信、低开销）。

### 4. 智能化：AI + 大模型集成 (Intelligence Layer)
* **内容**：展示 AI 如何从简单的 `if-else` 进化为 **LLM 语义理解**。
    * 展示一个对话案例：用户说“我感觉有点闷”，AI 结合 Node 1 的 CO2 数据（>1000ppm）自动决策开启风扇。
* **专家技巧**：提到 **Context-Aware (上下文感知)**。AI 不只是聊天，而是基于传感器状态做决策。

### 5. 数据驱动：MATLAB Analysis (Data Layer) —— **[Grade A 关键标志]**
* **内容**：展示你在 ThingSpeak 上的图表。
    * **对比图**：展示传感器原始噪声数据 vs 经过 MATLAB **Hampel Filter (去噪)** 后的平滑数据。
    * **分析结果**：通过 MATLAB 计算出的每日环境质量评分。
* **专家技巧**：引用 Activity 10 的内容。强调**数据清洗 (Data Cleaning)** 对 IoT 可靠性的重要性。

### 6. UX 设计与安全保障 (UX & Security)
* **UX**：展示多模态交互（手势控制、网页中控、硬件屏幕反馈）。引用 L06 的交互准则。
* **Security**：提到 API Key 的保护、Node 3 的生物识别安全性。引用 L07 的 **Threat Modeling**。

## 💡 汇报时的“加分”话术（专家级表达）

1.  **谈及协议时**：
    > “在设计 L-CASH 时，我没有选择 HTTP 作为设备通信协议，而是采用了 **MQTT**。因为在资源受限的 M5Stack 节点上，MQTT 的轻量级头部和发布/订阅机制能显著降低网络延迟并节省能源，这符合 L04 中提到的协议选择原则。”

2.  **谈及数据处理时**：
    > “我们意识到传感器在现实环境中存在随机噪声。因此，我利用 **MATLAB Analysis** 在云端实现了实时去噪。这确保了后端 AI 接收到的是可靠的物理信号，避免了系统误触发，提升了整体鲁棒性。”

3.  **谈及未来扩展（Blockchain）**：
    > “考虑到未来隐私保护的需求，我们可以参考 L08 引入 **Blockchain** 技术，将 Node 3 的门禁事件记录在不可篡改的账本上，从而解决智能家居的信任问题。”
