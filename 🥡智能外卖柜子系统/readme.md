# 📦 智能外卖柜模块 (Smart Takeout Cabinet - M5GO)

本项目是智慧城市 AIoT 系统中的**物理执行终端**，基于 M5GO 开发，集成了生物识别、环境感知与自动化控制。它负责将系统的“逻辑决策”转化为“物理动作”，并实时向中控反馈运行状态。



---

## 🛠️ 核心技术架构 (Tech Stack)
* **硬件平台**: M5GO (ESP32 主控)
* **感知单元**: 
    * FINGER Unit (指纹识别 - 安全认证)
    * TOF Unit (激光测距 - 环境感知)
* **执行单元**: Servo (舵机锁扣 - 自动化开锁)
* **通信协议**: MQTT (Pub/Sub) 协议接入 Emqx 消息总线

---

## ⚙️ 系统逻辑实现 (Key Logic)

### 1. 原子化动作与安全锁机制 (Mutex Lock)
为了防止并发指令冲突或硬件误操作，模块内置了 `is_working` 互斥锁。
* **逻辑说明**：当系统处于执行序列中（如舵机转动、柜门开合等待期），`is_working` 被置为 `True`，此时硬件将屏蔽所有后续的 MQTT 指令或指纹输入，确保动作序列的完整性与安全性。

### 2. 多模态指令触发
支持三种触发机制，满足不同交互场景：
* **生物识别触发**：指纹识别成功后执行 `open_door_sequence`。
* **远程遥控触发**：通过 MQTT 订阅 `city/device_A/cmd`，接收中控(同学D)或 AI 智能体指令。
* **本地调试**：支持通过物理按键直接进行用户指纹录入管理。

### 3. 环境与状态反馈
* **实时监控**：TOF 激光传感器实时监测柜体前方 300mm 范围内的环境变化，并在 LCD 上反馈状态（Warning/Welcome）。
* **异步汇报**：模块以 5 秒/次的频率通过 MQTT 周期性向中控上报设备状态，确保“中控-终端”状态的一致性。

## 💻 关键代码逻辑说明 (`finger+TOF+servo+web.py`)

```python
# 核心动作序列：保证操作的原子性
def open_door_sequence(name, mode='Finger'):
    global is_working
    is_working = True  # 锁定状态，屏蔽外部干扰
    servo_0.angle(90)   # 舵机开锁
    # LCD 与 RGB 反馈...
    time.sleep(5)       # 等待存取
    servo_0.angle(0)    # 自动归位关锁
    is_working = False  # 释放锁
```

## 🚀 联调与部署说明
1. **MQTT 配置**：确保 M5GO 与中控系统处于同一 MQTT Broker (`broker.emqx.io`) 下。
2. **Topic 规范**：
    * 状态上报: `city/status/a`
    * 控制接收: `city/device_A/cmd`
3. **安全协同**：在系统全局出现 `emergency: True` (如火警) 时，模块应通过软件逻辑自动禁用物理接口，以符合消防应急规范。
