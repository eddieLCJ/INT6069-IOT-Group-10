# 🚒 智慧应急响应子系统 (Fire Emergency Response Subsystem)

## 📖 项目概述
本项目是智慧城市 MQTT 分布式协同系统的**安全预警核心节点**。由同学 B 负责开发，主要功能是实时监测环境安全状态，并在检测到火情预警时，通过 MQTT 协议向全城广播应急信号，触发跨设备联动（如锁定外卖柜、开启路政照明、AI 避险提示）。

本子系统采用 **M5GO** 硬件平台与 **UIFlow** 图形化编程实现，具备高实时性、声光双重报警以及全局状态同步能力。

## 📡 通信协议 (Communication Protocol)
作为系统的“安全哨兵”，本模块负责发布最高优先级的应急指令：

* **广播主题 (Publish Topic)**: `city/status/b`
* **应急负载 (Payload)**: 
    * `正常`: 环境参数在安全阈值内。
    * `FIRE_ALARM`: 检测到温度异常，触发全局火警。
* **中控联动**: 同学 D 的中控系统在接收到 `FIRE_ALARM` 后，会将全局变量 `emergency` 设为 `True`。

---

## ⚙️ 核心功能逻辑
系统的运行遵循 **“采集 -> 判定 -> 广播 -> 联动”** 的闭环路径：

1.  **数据采集**: ENV 传感器每 500ms 采集一次当前的温度值。
2.  **火情判定**: 
    * 设定温度阈值（如 30°C - 35°C）。
    * 若 `Current_Temp > Threshold`，系统进入“火灾应急模式”。
3.  **本地报警**: 
    * **视觉**: M5GO 灯条触发红色爆闪逻辑。
    * **听觉**: 内置扬声器循环播放报警音效。
    * **显示**: LCD 屏幕切换为红色预警界面。
4.  **远程广播**: 每 2 秒向 MQTT 主题 `city/status/b` 发送一次报警信号，确保系统其他节点（中控、外卖柜、AI）能稳定接收。

## 🚀 部署与测试

### 1. 软件环境
* 使用 **UIFlow 1.x** 在线或离线编辑器。
* 配置 MQTT 参数：Host 为 `broker.emqx.io`，端口 `1883`。

### 2. 本地测试
* **正常模式**: RGB 灯显示绿色，屏幕实时显示温湿度。
* **模拟测试**: 使用温热物体靠近 ENV 传感器，观察温度达标后是否立即触发红色爆闪与报警音。
* **自动解除**: 温度降低后，确认报警自动停止，系统恢复“正常”状态。

### 3. 系统联调
* 触发火警后，检查同学 D 的中控面板是否变红。
* 验证同学 A 的外卖柜是否因接收到火警信号而自动锁定。

## 🏆 项目亮点
* **高优先级保障**: 应急信号具备全系统“一票否决权”，是智慧城市运行的安全底座。
* **声光联动控制**: 提供了直观的人机交互反馈，模拟真实的逃生指引逻辑。
* **分布式协同**: 完美适配 MQTT 通信协议，实现了跨硬件（M5GO 到 ESP32-S3）的无缝联动。
